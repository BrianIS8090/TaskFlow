# Инструкция по обновлению и деплою TaskFlow

Этот документ описывает, как собрать новую версию приложения и отправить её на сервер.

## 1. Подготовка

Убедитесь, что вы находитесь в папке проекта:
```powershell
cd taskflow
```

## 2. Разработка (Локальный запуск)

Если вы внесли изменения в код и хотите проверить их локально:
```powershell
npm run dev
```
Откройте ссылку, которую покажет терминал (обычно `http://localhost:5173`).

## 3. Сборка (Build)

Когда изменения проверены и вы готовы к публикации, нужно собрать "релизную" версию. Эта команда создаст папку `dist` с оптимизированными файлами.

```powershell
npm run build
```

*После этой команды в папке `taskflow/dist` появится готовый сайт (HTML, CSS, JS).*

## 4. Деплой (Публикация)

### Вариант А: Firebase Hosting (Настроено по умолчанию)
Используйте этот метод, чтобы обновить сайт по адресу `https://taskflow-app-brian-01.web.app`.

Выполните команду:
```powershell
npx firebase deploy --only hosting
```

---

### Вариант Б: Сторонний хостинг (TimeWeb / АПМО / Любой статический сервер)
Если вы используете другую платформу (например, TimeWeb App Platform, Nginx, Apache), вам нужна только папка `dist`.

1. Выполните сборку: `npm run build`
2. Зайдите в папку `taskflow/dist`.
3. Скопируйте **все файлы** из этой папки.
4. Загрузите их на ваш сервер (через FTP, панель управления или Git) в публичную директорию (обычно `public_html` или корневую папку сайта).

**Важно:** Для работы приложения серверу не нужен Node.js. Ему нужно просто отдавать статические файлы.

## Решение проблем

*   **Ошибка "permission-denied":**
    Проверьте правила безопасности в `firestore.rules` и убедитесь, что они опубликованы:
    `npx firebase deploy --only firestore:rules`

*   **Ошибка "index required":**
    Откройте консоль браузера, найдите ссылку на создание индекса и перейдите по ней.
